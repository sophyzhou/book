Data Models

The data model for this project would require 3 types of data.

  • Users
  • Movies
  • Chatting Rooms

The data model structure is shown as below:

  • Users

    • key - user
      • username
      • email
      • imageURL

  • Movies

    • key - movie
      • title
      • YouTubeID
      • year
      • review

  • Chatrooms

    • key - users
      • totalCount
      • key - user1
        • username
        • imageURL
      • key - user2
        • username
        • imageURL
    • key - record
      • key - record1
        • username
        • text
      • key - record2
        • username
        • text

Revise

  • The client should be able to have awareness of what every other user is doing concurrently. The client could see who is typing right now. The client can also see other clients status (like enter the chat room, leave the chat room) in the message box below the chatting box.
